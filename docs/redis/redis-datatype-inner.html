Redis数据类型内部实现机制详解

深入探讨Redis各数据类型的底层实现原理与编码方式

1. Redis对象系统(RedisObject)

typedef struct redisObject {
    unsigned type:4;    // 类型: STRING, LIST, HASH, SET, ZSET
    unsigned encoding:4;// 编码方式
    void *ptr;         // 指向实际数据结构的指针
    int refcount;      // 引用计数
    unsigned lru:24;   // LRU时间或LFU数据
} robj;

编码类型(encoding)

  • OBJ_ENCODING_RAW: 原始编码
  • OBJ_ENCODING_INT: 整数编码
  • OBJ_ENCODING_HT: 哈希表编码
  • OBJ_ENCODING_ZIPMAP: 压缩映射
  • OBJ_ENCODING_LINKEDLIST: 双端链表
  • OBJ_ENCODING_ZIPLIST: 压缩列表
  • OBJ_ENCODING_INTSET: 整数集合
  • OBJ_ENCODING_SKIPLIST: 跳跃表
  • OBJ_ENCODING_EMBSTR: 嵌入式字符串

对象类型(type)

  • OBJ_STRING: 字符串对象
  • OBJ_LIST: 列表对象
  • OBJ_SET: 集合对象
  • OBJ_ZSET: 有序集合对象
  • OBJ_HASH: 哈希对象

2. 基础数据结构实现

2.1 SDS (Simple Dynamic String)

struct sdshdr {
    uint32_t len;    // 字符串长度
    uint32_t free;   // 未使用空间长度
    char buf[];      // 字符数组
};

实现特点：

  • O(1)时间复杂度获取字符串长度
  • 避免缓冲区溢出
  • 减少修改字符串时的内存重分配次数
  • 二进制安全
  • 兼容部分C字符串函数

2.2 压缩列表(ziplist)

graph LR A[zlbytes] --> B[zltail] B --> C[zllen] C --> D[entry1] D --> E[entry2] E --> F[...] F --> G[entryN] G --> H[zlend]

组成部分：

  • zlbytes: 整个ziplist占用字节数
  • zltail: 最后一个entry的偏移量
  • zllen: entry个数
  • entry: 具体节点
  • zlend: 结束标记符

2.3 跳跃表(skiplist)

typedef struct zskiplistNode {
    sds ele;                  // 元素值
    double score;            // 分数
    struct zskiplistNode *backward;    // 后退指针
    struct zskiplistLevel {
        struct zskiplistNode *forward; // 前进指针
        unsigned long span;           // 跨度
    } level[];              // 层
} zskiplistNode;

typedef struct zskiplist {
    struct zskiplistNode *header, *tail; // 表头节点和表尾节点
    unsigned long length;               // 节点的数量
    int level;                         // 层数最大的节点的层数
} zskiplist;

特性：

  • 平均O(logN)的查找复杂度
  • 支持范围查找操作
  • 允许重复的score值
  • 相比平衡树实现简单，内存占用小

2.4 字典(dict)

typedef struct dict {
    dictType *type;     // 类型特定函数
    void *privdata;     // 私有数据
    dictht ht[2];       // 哈希表(用于rehash)
    int rehashidx;      // rehash索引
    int iterators;      // 当前正在使用的迭代器数量
} dict;

特点：

  • 渐进式rehash
  • 负载因子触发扩容/缩容
  • 链地址法解决哈希冲突

3. 数据类型实现细节

3.1 String类型实现

编码方式：

  • int: 8个字节的长整型
  • embstr: ≤44字节的字符串
  • raw: >44字节的字符串
graph TD A[String] --> B{是否为整数?} B -->|是| C[int编码] B -->|否| D{长度≤44字节?} D -->|是| E[embstr编码] D -->|否| F[raw编码]

3.2 Hash类型实现

编码转换条件：

  • ziplist条件：
    • 所有键值对的键和值的字符串长度都小于64字节
    • 键值对数量小于512个
  • 否则使用hashtable编码

3.3 List类型实现

quicklist结构：

  • 结合ziplist和linkedlist的优点
  • 每个节点使用ziplist来保存数据
  • 节点之间使用双向指针连接

3.4 Set类型实现

编码选择：

  • intset条件：
    • 所有元素都是整数
    • 元素数量不超过512个
  • 否则使用hashtable编码

3.5 Sorted Set类型实现

实现方式：

  • ziplist编码条件：
    • 元素数量小于128个
    • 所有元素长度小于64字节
  • skiplist编码：
    • 同时使用skiplist和dict
    • skiplist用于元素排序
    • dict用于O(1)复杂度查找成员

4. 编码转换机制详解

4.1 编码转换触发条件

数据类型 源编码 目标编码 转换条件
String int raw 当数字无法用long表示时
String embstr raw 当字符串修改时
Hash ziplist hashtable 元素数量>512或任一元素>64字节
Set intset hashtable 元素数量>512或加入非整数元素
Zset ziplist skiplist 元素数量>128或任一元素>64字节

6. 性能优化机制

6.1 渐进式rehash

int dictRehash(dict *d, int n) {
    int empty_visits = n * 10;
    if (!dictIsRehashing(d)) return 0;
    
    while (n-- && d->ht[0].used != 0) {
        dictEntry *de, *nextde;
        while(d->ht[0].table[d->rehashidx] == NULL) {
            d->rehashidx++;
            if (--empty_visits == 0) return 1;
        }
        de = d->ht[0].table[d->rehashidx];
        while(de) {
            uint64_t h;
            nextde = de->next;
            h = dictHashKey(d, de->key) & d->ht[1].sizemask;
            de->next = d->ht[1].table[h];
            d->ht[1].table[h] = de;
            d->ht[0].used--;
            d->ht[1].used++;
            de = nextde;
        }
        d->ht[0].table[d->rehashidx] = NULL;
        d->rehashidx++;
    }
    return 1;
}

渐进式rehash特点：

  • 分多次执行rehash，避免阻塞
  • 同时保持两个hash表
  • 期间可以正常处理命令
  • 新的数据一律写入新表

6.2 压缩列表连锁更新优化

优化策略：

  • 预留空间策略
    • 插入时预留额外空间
    • 减少连锁更新概率
  • 设置合理的转换阈值
    • hash-max-ziplist-entries
    • hash-max-ziplist-value

7. 关键源码解析

7.1 跳跃表节点创建

zskiplistNode *zslCreateNode(int level, double score, sds ele) {
    zskiplistNode *zn = zmalloc(sizeof(*zn)+level*sizeof(struct zskiplistLevel));
    zn->score = score;
    zn->ele = ele;
    return zn;
}

zskiplist *zslCreate(void) {
    int j;
    zskiplist *zsl;

    zsl = zmalloc(sizeof(*zsl));
    zsl->level = 1;
    zsl->length = 0;
    zsl->header = zslCreateNode(ZSKIPLIST_MAXLEVEL,0,NULL);
    for (j = 0; j < ZSKIPLIST_MAXLEVEL; j++) {
        zsl->header->level[j].forward = NULL;
        zsl->header->level[j].span = 0;
    }
    zsl->header->backward = NULL;
    zsl->tail = NULL;
    return zsl;
}

7.2 字典扩容触发

static int _dictExpandIfNeeded(dict *d) {
    if (dictIsRehashing(d)) return DICT_OK;

    if (d->ht[0].size == 0) return dictExpand(d, DICT_HT_INITIAL_SIZE);

    if (d->ht[0].used >= d->ht[0].size &&
        (dict_can_resize ||
         d->ht[0].used/d->ht[0].size > dict_force_resize_ratio))
    {
        return dictExpand(d, d->ht[0].used*2);
    }
    return DICT_OK;
}
