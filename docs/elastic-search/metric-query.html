Elasticsearch聚合查询之Metric聚合详解

Elasticsearch作为一个强大的分布式搜索和分析引擎，其聚合功能是数据分析的核心。本文将深入探讨Metric聚合，帮助您掌握如何使用这些聚合来计算各种统计指标。

前置知识：本文假设您已经了解Elasticsearch的基本概念和查询语法。如果您是初学者，建议先阅读Elasticsearch的基础教程。

2. Metric聚合概述

Metric聚合是Elasticsearch中用于计算数值型字段统计信息的聚合类型。它们可以计算诸如平均值、最大值、最小值等指标，是数据分析中不可或缺的工具。

graph TD A[Metric聚合] --> B[单值指标聚合] A --> C[多值指标聚合] B --> D[avg - 平均值] B --> E[min - 最小值] B --> F[max - 最大值] B --> G[sum - 求和] B --> H[value_count - 计数] B --> I[cardinality - 基数] C --> J[stats - 基础统计] C --> K[extended_stats - 扩展统计] C --> L[percentiles - 百分位] C --> M[percentile_ranks - 百分位等级] C --> N[matrix_stats - 矩阵统计] C --> O[geo_bounds - 地理边界] C --> P[geo_centroid - 地理中心点] C --> Q[top_metrics - 顶部指标]

Metric聚合主要分为两类：

  • 单值指标聚合（Single-value metrics aggregations）：生成单个值的聚合，如avg、min、max等
  • 多值指标聚合（Multi-value metrics aggregations）：生成多个值的聚合，如stats、extended_stats、percentiles等

基本语法结构：

{
  "aggs": {
    "聚合名称": {
      "聚合类型": {
        "field": "字段名"
      }
    }
  }
}

3. 单值指标聚合

单值指标聚合返回单个值作为聚合结果。这些聚合对于快速了解数据的基本特征非常有用。

3.1 平均值聚合（avg）

计算指定字段的平均值。

GET /sales/_search
{
  "size": 0,
  "aggs": {
    "avg_price": {
      "avg": {
        "field": "price"
      }
    }
  }
}

响应示例：

{
  ...
  "aggregations": {
    "avg_price": {
      "value": 25.5
    }
  }
}

3.2 最小值聚合（min）

7.3 百分位近似算法

Elasticsearch使用TDigest算法来近似计算百分位值，可以通过compression参数控制精度。

GET /sales/_search
{
  "size": 0,
  "aggs": {
    "price_percentiles": {
      "percentiles": {
        "field": "price",
        "percents": [1, 5, 25, 50, 75, 95, 99],
        "tdigest": {
          "compression": 200
        }
      }
    }
  }
}

说明：compression参数越大，准确性越高，但内存消耗也越大。默认值是100。

7.4 HDR直方图百分位

除了TDigest算法外，Elasticsearch还支持高动态范围（HDR）直方图算法。

GET /sales/_search
{
  "size": 0,
  "aggs": {
    "price_percentiles": {
      "percentiles": {
        "field": "price",
        "percents": [1, 5, 25, 50, 75, 95, 99],
        "hdr": {
          "number_of_significant_value_digits": 3
        }
      }
    }
  }
}

提示：HDR算法在处理具有已知范围和精度要求的数据时表现很好，但可能不适合所有情况。

8. 基数聚合详解

基数聚合用于计算字段的唯一值数量，类似于SQL中的COUNT(DISTINCT)。它使用HyperLogLog++算法来提供近似但高效的计算。

8.1 基本用法 

GET /sales/_search
{
  "size": 0,
  "aggs": {
    "unique_customers": {
      "cardinality": {
        "field": "customer_id"
      }
    }
  }
}

响应示例：

{
  ...
  "aggregations": {
    "unique_customers": {
      "value": 85
    }
  }
}

8.2 精度控制

可以通过precision_threshold参数控制基数聚合的精度。

GET /sales/_search
{
  "size": 0,
  "aggs": {
    "unique_customers": {
      "cardinality": {
        "field": "customer_id",
        "precision_threshold": 100
      }
    }
  }
}

说明：precision_threshold参数决定了用于计算的内存。当唯一值数量少于此阈值时，计算结果是精确的；超过此阈值，结果是近似的。最大值为40000，默认值为3000。

8.3 实际应用场景

基数聚合的典型应用场景包括：

  • 计算网站的独立访问者数量
  • 分析不同产品的购买用户数
  • 监控系统中的唯一错误类型数量
  • 计算每天的活跃用户数
GET /logs/_search
{
  "size": 0,
  "aggs": {
    "daily_active_users": {
      "date_histogram": {
        "field": "timestamp",
        "calendar_interval": "day"
      },
      "aggs": {
        "unique_users": {
          "cardinality": {
            "field": "user_id"
          }
        }
      }
    }
  }
}

9. 脚本指标聚合

脚本指标聚合允许使用自定义脚本来计算复杂的指标。这提供了极大的灵活性，可以实现标准聚合无法满足的需求。

9.1 基本结构

脚本指标聚合包含四个阶段的脚本：

  1. init_script：初始化脚本，在聚合开始前执行
  2. map_script：对每个文档执行的脚本
  3. combine_script：在每个分片上执行，组合map_script的结果
  4. reduce_script：汇总所有分片的结果
GET /sales/_search
{
  "size": 0,
  "aggs": {
    "sales_script": {
      "scripted_metric": {
        "init_script": "state.transactions = []",
        "map_script": "state.transactions.add(doc.price.value * doc.quantity.value)",
        "combine_script": "double sum = 0; for (t in state.transactions) { sum += t } return sum",
        "reduce_script": "double sum = 0; for (a in states) { sum += a } return sum"
      }
    }
  }
}

9.2 实际示例

以下示例计算每个订单中价格最高的3个商品的总和：

GET /sales/_search
{
  "size": 0,
  "aggs": {
    "top3_sales": {
      "scripted_metric": {
        "init_script": "state.prices = []",
        "map_script": "state.prices.add(doc.price.value)",
        "combine_script": """
          state.prices.sort(Comparator.reverseOrder());
          double sum = 0;
          int i = 0;
          for (price in state.prices) {
            if (i >= 3) break;
            sum += price;
            i++;
          }
          return sum;
        """,
        "reduce_script": "double sum = 0; for (a in states) { sum += a } return sum"
      }
    }
  }
}

警告：脚本指标聚合功能强大但可能影响性能。对于复杂计算或大数据集，请谨慎使用。

9.3 使用参数

您可以通过params参数向脚本传递参数，使脚本更加灵活：

GET /sales/_search
{
  "size": 0,
  "aggs": {
    "sales_script": {
      "scripted_metric": {
        "init_script": "state.transactions = []",
        "map_script": "state.transactions.add(doc.price.value * doc.quantity.value * params.tax_multiplier)",
        "combine_script": "double sum = 0; for (t in state.transactions) { sum += t } return sum",
        "reduce_script": "double sum = 0; for (a in states) { sum += a } return sum",
        "params": {
          "tax_multiplier": 1.1
        }
      }
    }
  }
}

10. 管道指标聚合

管道聚合对其他聚合的输出进行操作，而不是直接处理文档。它们可以用于进一步处理聚合结果。

graph LR A[文档] --> B[普通聚合] B --> C[管道聚合] C --> D[最终结果]

10.1 管道聚合类型

管道聚合有多种类型，包括：

  • Parent：对父聚合的结果进行操作，如derivative、moving_avg
  • Sibling：创建与其他聚合同级的结果，如cumulative_sum、stats_bucket

10.2 移动平均聚合

          "moving_avg": {
            "buckets_path": "sales",
            "window": 3,
            "model": "simple"

  }

11. 最佳实践

11.1 性能优化

  • 使用docvalue_fields：确保聚合字段启用了docvalues以提高性能
  • 限制搜索结果：当只需要聚合结果时，将size设为0
  • 使用过滤器：在聚合前使用过滤器减少处理的文档数量
  • 避免脚本滥用：尽量使用内置聚合而不是脚本聚合
  • 控制基数：对高基数字段进行聚合时要小心，考虑使用terms聚合的size参数
GET /sales/_search
{
  "size": 0,
  "query": {
    "range": {
      "date": {
        "gte": "now-1y"
      }
    }
  },
  "aggs": {
    "sales_stats": {
      "stats": {
        "field": "price"
      }
    }
  }
}

11.2 组合聚合

组合多种聚合类型可以提供更全面的数据分析：

        "field": "category",

        "monthly_sales": {
          "date_histogram": {
            "field": "date",
            "calendar_interval": "month"
          },
          "aggs": {
            "sales": {
              "sum": {
                "field": "price"
              }
            },
            "sales_trend": {
              "moving_avg": {
                "buckets_path": "sales",
                "window": 3

            }

        },
        "category_stats": {
          "stats": {
            "field": "price"
          }

                  "order": "desc"
                }

    }
  }
}

12. 总结

Elasticsearch的Metric聚合提供了强大的数据分析能力，从简单的统计计算到复杂的脚本操作和管道聚合，满足了各种数据分析需求。

本文详细介绍了各种Metric聚合类型及其应用场景：

  • 单值指标聚合（avg、min、max等）提供基本统计信息
  • 多值指标聚合（stats、extended_stats等）提供更全面的统计分析
  • 百分位聚合帮助理解数据分布
  • 基数聚合计算唯一值数量
  • 脚本指标聚合支持自定义计算逻辑
  • 管道聚合进一步处理聚合结果

通过合理组合这些聚合，可以构建复杂的分析系统，从海量数据中提取有价值的信息，支持业务决策和系统监控。

在使用Metric聚合时，要注意性能优化和资源管理，特别是对于大数据量和高基数字段的情况。合理使用过滤器、限制结果大小，选择合适的聚合类型，将帮助您获得最佳的Elasticsearch性能和分析体验。
